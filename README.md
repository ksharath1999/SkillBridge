# ⚙️ Local Setup

## 1. Clone repo

git clone <your-repo-url>  
cd skillbridge-api  

---

## 2. Create virtual environment

python -m venv venv  
venv\Scripts\activate  

---

## 3. Install dependencies

pip install -r requirements.txt  

---

## 4. Setup environment variables

Create `.env` file:

DATABASE_URL=your_neon_db_url  
SECRET_KEY=your_secret  
MONITORING_API_KEY=12345  

---

## 5. Run app (creates tables)

uvicorn src.main:app --reload  

---

## 6. Seed database

python -m src.seed  

---

# 🔐 Authentication Flow

## Login

curl -X POST https://your-deployed-url.onrender.com/auth/login \
-H "Content-Type: application/json" \
-d '{
  "email": "trainer0@test.com",
  "password": "1234"
}'

👉 Response:

{
  "access_token": "..."
}

---

## Use Token

Authorization: Bearer <token>

# 📌 API Endpoints

## Auth
- POST /auth/signup  
- POST /auth/login  

## Batch
- POST /batches  
- POST /batches/{batch_id}/invite  
- POST /batches/join  

## Sessions
- POST /sessions  

## Attendance
- POST /attendance/mark  

## Monitoring
- GET /monitoring/attendance  
- POST /monitoring/attendance → returns 405  

## Summary
- GET /batches/{batch_id}/summary  
- GET /programme/summary

# 🧠 Schema Decisions

### batch_students
Used for mapping students to batches.

### batch_invites
Token-based join system:
- generated by trainer  
- expires  
- single-use  

### Attendance
Unique constraint on:
(session_id, student_id)

Prevents duplicate attendance entries.

### Monitoring Officer
Uses API key-based authentication instead of JWT.

---

# ⚠️ Validation & Error Handling

- 422 → invalid input  
- 404 → missing resources  
- 403 → unauthorized access  
- 401 → missing/invalid token  
- 405 → invalid method on monitoring endpoint
